## 一、业务专业术语

| 术语 | 定义 |
|:-----|:-----|
| 多租户 | 一种软件架构模式，多个租户共享同一个系统实例，但数据完全隔离。在本系统中，每个店铺为一个租户，拥有独立的数据。 |
| 积分商城 | 以积分作为唯一货币的商品兑换系统，用户通过积累的积分来兑换商品，不支持现金支付。 |
| 店铺用户 | 每个店铺的管理员账号，负责本店的商品管理、订单处理、用户积分维护等运营工作。账号与店铺是独立概念。 |
| 普通用户 | 店铺的消费者，通过积分兑换商品的用户群体。每个普通用户只属于一个店铺。 |
| 商家自定义分类 | 店铺用户自行创建的商品分类，用于对本店商品进行分组管理。 |
| 系统内置分类 | 系统预置的商品分类（如：虚拟、实物等），店铺用户不可编辑，仅可选择。 |
| 逻辑删除 | 一种数据删除方式，不是真正从数据库中删除数据，而是通过修改状态字段（如标记为"禁用"或"已删除"）使数据不可见。 |
| 表级别隔离 | 多租户数据隔离的一种实现方式，所有租户的数据存在同一张表中，通过 shop_id 字段区分数据归属。 |

## 二、业务规则术语

| 术语 | 定义 |
|:-----|:-----|
| 单用户最大兑换个数 | 商品的可重复兑换限制设置。默认0表示无限制，非0表示同一用户对该商品的最高兑换次数。 |
| 积分操作记录 | 店铺用户增加或扣除普通用户积分时，必须填写的操作原因说明，用于积分变动的可追溯。 |
| 订单超时提醒 | 订单状态为「已下单」超过7天未确认时，系统自动发送提醒通知店铺用户处理。 |
| 店铺冻结 | 管理员对店铺执行的强制封停操作。店铺冻结后，店铺用户无法操作商品和用户，商品自动下架。 |
| 邀请码 | 用于替代直接注册/创建流程的安全凭证。唯一对应一个目标账号类型（店铺用户/普通用户），生成后永久有效，重新生成后旧码立即失效。 |
| 强制重置密码 | 管理员或店铺用户对用户账号执行的密码重置操作。系统随机生成新密码，弹框显示后由执行者线下告知用户。 |

## 三、系统技术术语

| 术语 | 定义 |
|:-----|:-----|
| shop_id | 表级别隔离中用于标识数据所属店铺的字段。通过 shop_id 过滤确保店铺间数据完全隔离。 |
| 卡片/表格切换 | 商品列表和订单列表的两种展示形式切换功能。用户可自由选择，系统记住用户偏好。 |
| 收货信息 | 买家在兑换实物商品时填写的收货信息，包括收货人姓名、手机号码、收货地址。手机号用于物流联系。 |

## 四、状态术语

| 术语 | 定义 |
|:-----|:-----|
| 账号状态-待审核 | 店铺用户或普通用户已提交注册申请，等待管理员或店铺用户审核的状态。审核中的用户尝试登录时系统提示"正在审核中"。 |
| 账号状态-已通过 | 账号注册申请已审核通过，账号可正常登录使用的状态。 |
| 账号状态-已拒绝 | 账号注册申请被拒绝，账号无法登录，但可重新填写信息再次注册的状态。 |
| 店铺状态-未申请 | 店铺用户刚注册账号但尚未申请店铺的状态。 |
| 店铺状态-申请中 | 店铺用户已提交店铺申请，等待管理员审核的状态。 |
| 店铺状态-已通过 | 管理员审核通过，店铺可以正常运营的状态。 |
| 店铺状态-已拒绝 | 店铺申请被拒绝，店铺用户可重新填写理由再次申请的状态。 |
| 店铺状态-已冻结 | 管理员对店铺执行冻结操作后的状态，店铺功能受限。 |
| 订单状态-已下单 | 用户已提交订单，积分已扣除，等待商家确认的状态。 |
| 订单状态-已确认 | 商家已确认订单，等待发货的状态。 |
| 订单状态-已发货 | 商家已发货，商品配送中的状态。 |
| 订单状态-已完成 | 商家确认收货或用户确认收货后，订单完结的状态。 |
| 订单状态-已关闭 | 订单被用户或商家关闭后的状态，积分已返还用户。 |
