## 二、场景详细说明

### S01 店铺用户账号注册

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 店铺用户提交注册申请，管理员审核通过后账号生效；或通过管理员生成的邀请码注册 |
| 前置条件 | 店铺用户尚未注册或注册申请被拒绝 |
| 关键节点 | 注册申请提交, 管理员审核, 审核通过/拒绝 |
| 后置状态 | 账号状态变为已通过（可登录）或已拒绝 |

**分支场景**：
- 注册申请被拒绝后，店铺用户可重新填写信息再次申请
- **邀请码注册（管理员分发）**：管理员生成邀请码 → 分发给店铺用户 → 店铺用户使用邀请码注册 → 管理员审核 → 通过/拒绝

---

### S02 店铺申请管理

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 店铺用户申请开通店铺，管理员审核通过后店铺可正常运营；管理员可对店铺进行冻结/解冻操作 |
| 前置条件 | 店铺用户账号已审核通过 |
| 关键节点 | 填写店铺信息, 提交申请, 管理员审核, 审核通过/拒绝, 冻结, 解冻 |
| 后置状态 | 店铺状态：已通过 / 已拒绝 / 已冻结 / 申请中 / 未申请 |

**分支场景**：
- 申请被拒绝：店铺用户可重新填写理由再次申请
- 店铺冻结：店铺用户需重新申请审核，商品自动下架（已下单订单继续处理）
- 店铺解冻：恢复店铺正常运营状态

---

### S03 普通用户创建

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 店铺用户通过邀请码或直接创建的方式建立普通用户账号 |
| 前置条件 | 店铺状态为「已通过」 |
| 关键节点 | 邀请码生成, 分发邀请码, 普通用户注册, 审核, 直接创建, 批量创建, 密码生成 |
| 后置状态 | 普通用户账号生效（审核通过或直接创建） |

**分支场景**：
- 邀请码注册：普通用户注册后需店铺用户审核
- 直接创建：系统随机生成密码，店铺用户线下分发
- **重新生成邀请码**：新码生成后旧码立即失效，已在注册/审核中的普通用户账号继续可用

---

### S04 商品兑换

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 普通用户浏览商品列表，选择商品并使用积分兑换，填写收货信息后提交订单 |
| 前置条件 | 普通用户已登录，商品已上架 |
| 关键节点 | 浏览商品, 切换展示形式, 选择商品, 检查库存, 检查兑换次数, 填写收货信息（实物）, 提交订单, 积分扣除, 库存扣减 |
| 后置状态 | 订单状态变为「已下单」 |

**分支场景**：
- 库存不足：提示库存不足，无法下单
- 兑换次数超限：提示已达兑换上限，无法下单
- 虚拟商品：无需填写收货信息

---

### S05 订单处理

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 店铺用户对普通用户的订单进行确认、发货、确认完成等处理 |
| 前置条件 | 存在状态为「已下单」的订单 |
| 关键节点 | 查看订单, 确认订单, 填写快递类型, 填写快递单号, 发货, 确认完成 |
| 后置状态 | 订单状态依次变为「已确认」→「已发货」→「已完成」 |

**分支场景**：
- 7天超时提醒：已下单订单超过7天未处理，系统发送超时提醒
- 订单完成后：积分从用户账户正式扣除

---

### S06 订单关闭

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 订单在完成前被关闭，积分返还用户 |
| 前置条件 | 订单状态为「已下单」、「已确认」或「已发货」 |
| 关键节点 | 申请关闭, 填写关闭理由, 审核, 积分返还 |
| 后置状态 | 订单状态变为「已关闭」 |

**分支场景**：
- 已下单订单关闭：普通用户或店铺用户可申请关闭
- 已确认/已发货订单关闭：仅店铺用户可申请关闭
- 关闭后：积分返还用户账户

---

### S07 积分维护

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 店铺用户对普通用户的积分进行增加或扣除操作，需填写操作理由 |
| 前置条件 | 店铺状态为「已通过」，普通用户账号存在 |
| 关键节点 | 选择用户, 选择操作类型, 输入积分值, 填写操作理由, 确认操作, 记录变动, 更新余额 |
| 后置状态 | 用户积分余额更新，操作记录保存 |

**分支场景**：
- 增加积分：用户积分余额增加
- 扣除积分：用户积分余额减少

---

### S08 商品管理

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 店铺用户对本店商品进行全生命周期管理（创建、编辑、上架、下架、删除） |
| 前置条件 | 店铺状态为「已通过」 |
| 关键节点 | 创建商品, 编辑商品, 上架商品, 下架商品, 删除商品, 填写商品信息, 设置库存, 设置兑换限制 |
| 后置状态 | 商品状态更新或删除（逻辑删除） |

**商品信息字段**：名称, 图片, 积分价格, 库存, 商品分类（系统内置/商家自定义）, 描述, 单用户最大兑换个数

---

### S09 密码重置

| 属性 | 说明 |
|:-----|:-----|
| 场景描述 | 管理员或店铺用户重置用户密码，系统生成随机密码，线下告知用户 |
| 前置条件 | 管理员可重置任何用户；店铺用户仅可重置本店普通用户 |
| 关键节点 | 选择用户, 确认重置, 系统生成随机密码, 弹框显示密码, 线下告知 |
| 后置状态 | 用户密码已重置，新密码已生成 |

---

## 三、异常场景说明

| 异常场景 | 场景编号 | 处理方式 |
|:---------|:---------|:---------|
| 店铺冻结后操作受限 | S02, S07, S08 | 店铺冻结后，店铺用户不能操作商品、用户积分；商品自动下架 |
| 库存不足 | S04 | 提示库存不足，不允许下单 |
| 兑换次数超限 | S04 | 提示已达兑换上限，不允许下单 |
| 订单超时未处理 | S05 | 已下单订单超过7天未确认，系统发送超时提醒 |
| 审核拒绝 | S01, S02, S03 | 可重新填写信息再次申请/提交 |
