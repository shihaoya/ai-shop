## 一、干系人分析

### 1.1 平台投资方/企业主

| 属性 | 说明 |
|------|------|
| 干系人类型 | 投资人/管理层 |
| 关注点 | 项目投资回报率、系统建设成本控制、多店铺运营效率、市场竞争力提升 |

**具体关注点描述**：
- 系统建设成本与后期维护成本的平衡
- 多店铺统一管理带来的运营效率提升
- 积分体系设计的灵活性（支持各类营销活动）
- 数据统计报表的完整性和实时性，便于决策支持
- 系统可扩展性，能否支持业务快速增长

### 1.2 店铺运营团队（店铺用户）

| 属性 | 说明 |
|------|------|
| 干系人类型 | 终端用户（运营人员） |
| 关注点 | 商品上架效率、订单处理速度、用户积分管理便利性、店铺数据统计 |

**具体关注点描述**：
- 商品分类和上架流程的便捷性
- 订单状态的实时跟踪和处理效率
- 用户积分的手动维护（增加/扣除）操作简便
- 本店数据报表的查看（用户数、订单数、积分余额等）
- 店铺申请和审核流程的透明度

### 1.3 平台管理员

| 属性 | 说明 |
|------|------|
| 干系人类型 | 系统运营者 |
| 关注点 | 系统全局把控、店铺审核效率、跨店铺数据管理、账号安全 |

**具体关注点描述**：
- 全局仪表盘数据统计的实时性
- 店铺申请审核流程的高效处理
- 店铺冻结/解冻操作的便捷性
- 跨店铺用户账号管理的便捷性
- 密码重置等账号安全功能的可靠性

### 1.4 普通用户（消费者）

| 属性 | 说明 |
|------|------|
| 干系人类型 | 终端用户（消费者） |
| 关注点 | 商品浏览体验、兑换流程顺畅、订单状态透明、积分查询便利 |

**具体关注点描述**：
- 商品列表的展示形式（卡片/表格）切换便捷
- 商品搜索和分类查找的效率
- 下单流程的简洁性
- 订单状态的实时可查
- 个人积分余额的随时可见

## 二、角色分析

### 2.1 管理员（平台层）

| 属性 | 说明 |
|------|------|
| 系统中对应名称 | 管理员 |
| 角色层级 | 平台层 |

**使命描述**：
平台超级管理员是系统的最高权限拥有者，负责：
- 系统全局数据的监控和统计（所有店铺数量、用户总数、订单总量、积分发放统计）
- 店铺用户账号的审核和管理（审核店铺用户注册申请）
- 店铺的审核、冻结、解冻管理
- 跨店铺的普通用户账号管理
- 所有用户密码的强制重置

**权限边界**：
- 不直接管理积分（积分由店铺用户维护）
- 不直接操作店铺商品和订单

---

### 2.2 店铺用户（店铺层）

| 属性 | 说明 |
|------|------|
| 系统中对应名称 | 店铺用户 |
| 角色层级 | 店铺层 |

**使命描述**：
每个店铺的管理者，负责本店的具体运营：
- 店铺信息管理（申请店铺、查看店铺状态）
- 商品管理（创建/编辑/上架/下架/删除本店商品）
- 商品分类管理（创建/编辑/删除本店分类）
- 普通用户管理（创建用户、审核用户、查看用户列表）
- 积分维护（增加/扣除用户积分，填写操作理由）
- 订单处理（确认订单、发货、关闭订单）
- 密码重置（本店普通用户）

**权限边界**：
- 只能操作本店数据（商品、用户、订单、积分）
- 店铺状态为「申请中」或「已冻结」时，功能受限
- 店铺与账号是独立概念，需先有账号，再申请店铺

---

### 2.3 普通用户（用户层）

| 属性 | 说明 |
|------|------|
| 系统中对应名称 | 普通用户 |
| 角色层级 | 用户层 |

**使命描述**：
店铺的消费者，参与积分兑换的核心用户：
- 商品浏览（卡片/表格两种形式）
- 积分兑换下单（选择商品、填写收货信息）
- 订单管理（查看订单历史、关闭已下单订单）
- 个人中心（修改个人信息、查看积分余额）

**权限边界**：
- 只能访问所属店铺的数据
- 只能查看和操作自己的订单
- 只能查看自己的积分余额
- 积分由店铺用户手动维护，不可自行充值

## 五、关键说明

### 5.1 账号与店铺的分离

店铺用户账号与店铺是**独立概念**：
1. 店铺用户先注册账号 → 管理员审核通过
2. 店铺用户登录后 → 申请店铺 → 管理员审核通过
3. 店铺审核通过后 → 店铺用户才能上架商品

### 5.2 店铺状态对功能的影响

| 店铺状态 | 店铺用户可用功能 |
|----------|-----------------|
| 未申请 | 仅查看，不可操作商品 |
| 申请中 | 仅查看，不可操作商品 |
| 已通过 | 可正常管理商品、订单、用户积分 |
| 已冻结 | 无法操作，需要重新申请 |

### 5.3 数据隔离说明

- 管理员可查看所有店铺数据，但不可操作具体业务数据（商品、订单）
- 店铺用户只能操作本店数据
- 普通用户只能访问本店商品和查看个人订单
