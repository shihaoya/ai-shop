# 05_依赖分析

> **版本**：v1.0
> **生成时间**：2026-04-26

## 一、依赖概览

本系统为多店铺积分商城系统，属于**自建电商平台**，外部依赖较少，主要涉及以下几类：

| 依赖类别 | 依赖内容 | 依赖类型 | 必要性 |
|:---------|:---------|:---------|:-------|
| 消息通知服务 | 短信通知 | 能力依赖 | 必需（用于订单超时提醒等） |
| 文件存储服务 | 商品图片、店铺Logo等静态文件存储 | 能力依赖 | 必需（用于存储商品图片） |
| 邮件服务 | 邮件通知（可选） | 能力依赖 | 可选 |

> **说明**：本系统不涉及支付（积分兑换无需现金支付）、物流跟踪（快递单号由店铺用户手动填写，无须对接快递API）。

## 二、依赖关系矩阵

| 本系统功能 | 依赖的外部系统 | 依赖类型 | 依赖说明 |
|:---------|:-------------|:---------|:---------|
| 订单超时提醒 | 短信通知平台 | 能力依赖 | 需要发送短信通知店铺用户处理超时订单 |
| 商品图片上传 | 文件存储服务 | 能力依赖 | 需要存储商品主图、店铺Logo等图片文件 |

| 账号审核通知 | 短信/邮件通知平台 | 能力依赖 | 需要通知店铺用户账号审核结果（可选短信或邮件） |
| 密码重置通知 | 短信/邮件通知平台 | 能力依赖 | 需要通知用户密码已重置（可选短信或邮件） |

## 三、依赖详情

### 3.1 短信通知平台

| 属性 | 说明 |
|:-----|:-----|
| 依赖本系统的功能 | 订单超时提醒、账号审核结果通知、密码重置通知 |
| 依赖类型 | 能力依赖 |
| 依赖内容 | 需要调用短信平台的发送短信能力，向店铺用户发送通知消息 |
| 数据流向 | 本系统 → 短信平台 |

**依赖说明**：
- 订单超时提醒：店铺用户的订单超过7天未确认，系统自动发送短信提醒
- 账号审核通知：管理员审核店铺用户账号后，发送短信通知审核结果
- 密码重置通知：管理员重置用户密码后，发送短信通知用户

**短信模板示例**：
- 「【积分商城】尊敬的店铺用户，您有订单待处理，请尽快登录系统确认。」
- 「【积分商城】您的账号注册申请已通过审核，请登录系统。」

---

### 3.2 文件存储服务

| 属性 | 说明 |
|:-----|:-----|
| 依赖本系统的功能 | 商品管理、店铺管理 |
| 依赖类型 | 能力依赖 |
| 依赖内容 | 需要存储商品图片、店铺Logo等文件，并提供访问URL |
| 数据流向 | 本系统 → 文件存储服务 |

**依赖说明**：
- 商品图片：店铺用户上传商品主图，需要存储并生成访问URL
- 店铺Logo：店铺用户上传店铺Logo图片
- 用户头像：普通用户上传头像（如果有）

**存储需求**：
- 图片格式：支持 JPG、PNG、WebP 等常见格式
- 图片尺寸：商品主图建议 800x800，店铺Logo建议 200x200
- 访问方式：通过URL直接访问

---

### 3.3 邮件服务（可选）

| 属性 | 说明 |
|:-----|:-----|
| 依赖本系统的功能 | 账号审核通知、密码重置通知 |
| 依赖类型 | 能力依赖 |
| 依赖内容 | 需要发送邮件通知，作为短信通知的替代或补充 |
| 数据流向 | 本系统 → 邮件服务 |

**依赖说明**：
- 作为短信通知的备选方案
- 可用于发送更详细的通知内容（如包含链接的操作邮件）

---

## 四、无需对接的外部系统

根据需求分析，以下外部系统在当前阶段**无需对接**：

| 外部系统 | 无需对接原因 |
|:---------|:------------|
| 支付系统 | 积分兑换无需现金支付，无支付场景 |
| 物流跟踪API | 快递单号由店铺用户手动填写，无须自动跟踪物流状态 |
| 用户中心/SSO | 每个店铺独立管理用户，无需统一身份认证 |
| 商品中心 | 商品由店铺用户自行管理，无须对接外部商品库 |
| 营销系统 | 积分规则由系统内置，无须对接外部营销平台 |
