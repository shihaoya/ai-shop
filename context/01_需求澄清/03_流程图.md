## 流程分析

本系统包含以下核心业务流程：

| 序号 | 流程名称 | 流程类型 | 说明 |
|:----:|:---------|:---------|:-----|
| 1 | 店铺用户账号注册流程 | 简单流程 | 店铺用户提交注册，管理员审核 |
| 2 | 店铺申请流程 | 简单流程 | 店铺用户申请店铺，管理员审核 |
| 3 | 普通用户创建流程 | 简单流程 | 店铺用户生成邀请码或直接创建普通用户 |
| 4 | 商品兑换流程 | 简单流程 | 普通用户浏览并兑换商品 |
| 5 | 订单处理流程 | 简单流程 | 店铺用户确认订单、发货、确认完成 |
| 6 | 订单关闭流程 | 简单流程 | 用户或店铺用户关闭订单 |
| 7 | 积分维护流程 | CRUD流程 | 店铺用户增加/扣除普通用户积分 |
| 8 | 商品管理流程 | CRUD流程 | 店铺用户对商品进行增删改查 |
| 9 | 密码重置流程 | 简单流程 | 管理员或店铺用户重置用户密码 |
| 10 | 地址簿管理流程 | CRUD流程 | 普通用户新增/编辑/删除/设默认地址 |
| 11 | 地址簿选择流程 | 简单流程 | 实物兑换时从地址簿选择或手动输入地址 |

### 流程1：店铺用户账号注册流程

```mermaid
flowchart TD
    subgraph 店铺用户
        A([开始]) --> B[填写注册信息]
        B --> C[提交注册申请]
    end

    subgraph 管理员
        C --> D{账号审核}
        D -->|通过| E[开通账号登录权限]
        D -->|拒绝| F[拒绝申请]
    end

    subgraph 系统
        E --> G[账号生效]
        F --> H[通知店铺用户]
        G --> I([结束])
        H --> I
    end
```

**说明**：店铺用户注册账号后，需要管理员审核通过后才能登录。审核拒绝后店铺用户可重新填写理由再次申请。

### 流程2：店铺申请流程

```mermaid
flowchart TD
    subgraph 店铺用户
        A([开始]) --> B[登录系统]
        B --> C[填写店铺申请信息]
        C --> D[提交店铺申请]
    end

    subgraph 管理员
        D --> E{店铺审核}
        E -->|通过| F[开通店铺经营权]
        E -->|拒绝| G[拒绝申请]
    end

    subgraph 系统
        F --> H[店铺状态变为已通过]
        G --> I[通知店铺用户可重新申请]
        H --> J([结束])
        I --> J
    end

    style F fill:#90EE90
    style G fill:#FFB6C1
```

**说明**：
- 店铺申请通过后，店铺状态变为「已通过」，店铺用户可正常上架商品
- 店铺申请被拒绝后，店铺用户可重新填写理由再次申请
- 店铺冻结后，店铺用户需重新申请

### 流程3：普通用户创建流程

#### 3.1 邀请码注册方式

```mermaid
flowchart TD
    subgraph 店铺用户
        A([开始]) --> B[生成邀请码]
        B --> C[分发邀请码给消费者]
    end

    subgraph 普通用户
        C --> D[获取邀请码]
        D --> E[填写邀请码注册]
        E --> F[提交注册申请]
    end

    subgraph 店铺用户
        F --> G{用户审核}
        G -->|通过| H[开通账号]
        G -->|拒绝| I[拒绝申请]
    end

    subgraph 系统
        H --> J[普通用户账号生效]
        I --> K[通知消费者]
        J --> L([结束])
        K --> L
    end
```

#### 3.2 直接创建方式

```mermaid
flowchart TD
    subgraph 店铺用户
        A([开始]) --> B[直接创建用户账号]
        B --> C[设置初始积分]
        C --> D[确认创建]
    end

    subgraph 系统
        D --> E[系统随机生成密码]
        E --> F[账号生效]
    end

    subgraph 店铺用户
        F --> G[线下分发账号密码]
        G --> H([结束])
    end
```

**说明**：
- 邀请码注册方式：消费者通过邀请码注册后，需要店铺用户审核通过
- 直接创建方式：店铺用户批量创建账号，系统随机生成密码，线下分发

### 流程4：商品兑换流程

```mermaid
flowchart TD
    subgraph 普通用户
        A([开始]) --> B[浏览商品列表]
        B --> C[选择商品]
        C --> D{库存充足？}
        D -->|否| E[提示库存不足]
        D -->|是| F{兑换次数超限？}
        F -->|是| G[提示已达兑换上限]
        F -->|否| H[确认兑换]
    end

    subgraph 系统
        H --> I{商品类型}
        I -->|实物| J[填写收货信息]
        I -->|虚拟| K[跳过填写收货信息]
        J --> L[提交订单]
        K --> L
        L --> M[积分扣除]
        M --> N[库存扣减]
        N --> O[订单状态：已下单]
        O --> P([结束])
    end
```

**说明**：
- 商品兑换时检查：库存是否充足、用户兑换次数是否超限
- 实物商品需要填写收货信息（收货人姓名、手机号码、收货地址）
- 虚拟商品（电子券、点卡等）无需填写收货信息

### 流程5：订单处理流程

```mermaid
flowchart TD
    subgraph 普通用户
        A([开始]) --> B[下单]
        B --> C[订单状态：已下单]
    end

    subgraph 系统
        C --> D[7天超时提醒检测]
        D --> E[发送提醒给店铺用户]
    end

    subgraph 店铺用户
        E --> F[确认订单]
        F --> G[订单状态：已确认]
        G --> H[填写快递信息]
        H --> I[发货]
        I --> J[订单状态：已发货]
        J --> K[确认完成]
        K --> L[订单状态：已完成]
        L --> M([结束])
    end
```

**说明**：
- 订单「已下单」状态超过7天未确认，系统发送超时提醒
- 店铺用户发货时需要填写快递类型和快递单号
- 订单完成后状态变为「已完成」

### 流程6：订单关闭流程

```mermaid
flowchart TD
    subgraph 普通用户
        A([开始]) --> B[申请关闭订单]
        B --> C{订单状态}
        C -->|已下单| D[填写关闭理由]
    end

    subgraph 店铺用户
        C -->|已确认| E[申请关闭订单]
        C -->|已发货| E2[申请关闭订单]
        E --> G[填写关闭理由]
        E2 --> G
    end

    subgraph 系统
        D --> H[管理员/店铺用户审核]
        G --> H
        H -->|通过| I[积分返还用户]
        I --> J[订单状态：已关闭]
        J --> K([结束])
    end
```

**说明**：
- 已下单订单：普通用户可申请关闭，店铺用户可申请关闭
- 已确认/已发货订单：仅店铺用户可申请关闭
- 关闭订单必须填写理由（如：用户取消、商家缺货、其他）
- 关闭订单后积分返还用户

### 流程7：积分维护流程

```mermaid
flowchart TD
    subgraph 店铺用户
        A([开始]) --> B[选择操作类型]
        B --> C{增加积分 / 扣除积分}
        C -->|增加| D[输入增加积分值]

    end

    subgraph 系统
        F --> G[记录积分变动]
        G --> H[更新用户积分余额]
        H --> I([结束])
    end
```

**说明**：
- 店铺用户可以增加或扣除普通用户积分
- 增加和扣除操作都需要填写操作理由
- 操作记录用于积分变动的可追溯

### 流程8：商品管理流程

```mermaid
flowchart TD
    subgraph 店铺用户
        A([开始]) --> B[选择操作]
        B --> C[创建商品]
        B --> D[编辑商品]
        B --> E[上架商品]
        B --> F[下架商品]
        B --> G[删除商品]
    end

    subgraph 系统
        C --> H[填写商品信息]
        H --> I[名称、图片、积分、库存、分类]
        I --> J[设置单用户兑换限制]
        J --> K[保存商品]
        K --> L[商品状态：下架]

        D --> M[修改商品信息]
        M --> K

        E --> N[商品状态：上架]
        N --> L

        F --> O[商品状态：下架]
        O --> L

        G --> P[确认删除]
        P --> Q[逻辑删除标记]
        Q --> L

        L --> R([结束])
    end
```

**说明**：
- 创建商品时填写：名称、图片、积分价格、库存、商品基本信息（分类选择系统内置或商家自定义）、描述、单用户最大兑换个数
- 新创建商品默认为下架状态，需要手动上架
- 商品删除为逻辑删除，不是物理删除

### 流程9：密码重置流程

```mermaid
flowchart TD
    subgraph 管理员
        A([开始]) --> B[选择要重置的用户]
        B --> C[确认重置密码]
    end

    subgraph 店铺用户
        A2([开始]) --> B2[选择本店用户]
        B2 --> C2[确认重置密码]
    end

    subgraph 系统
        C --> D[系统生成随机密码]
        C2 --> D
        D --> E[弹框显示新密码]
        E --> F[管理员/店铺用户线下告知用户]
        F --> G([结束])
    end
```

**说明**：
- 管理员可以重置任何用户（店铺用户、普通用户）的密码
- 店铺用户只能重置本店普通用户的密码
- 新密码由系统随机生成，通过弹框显示，由管理员线下告知用户

### 流程10：地址簿管理流程

```mermaid
flowchart TD
    subgraph 普通用户
        A([开始]) --> B[进入个人中心]
        B --> C[进入地址簿]
        C --> D[查看地址列表]
        D --> E{选择操作}
        E -->|新增| F[填写地址信息]
        E -->|编辑| G[修改地址信息]
        E -->|删除| H[确认删除]
        E -->|设默认| I[确认设默认]
        F --> J[保存地址]
        G --> J
        H --> K[删除成功]
        I --> L[更新默认地址]
        J --> M{地址数<5？}
        M -->|是| D
        M -->|否| N[提示地址已满]
        K --> D
        L --> D
        N --> O([结束])
        D --> O
    end
```

**说明**：
- 普通用户最多管理5个收货地址
- 新增/编辑地址时需填写：收货人姓名、手机号码、收货地址
- 删除地址无限制（默认地址也可删除）
- 设置默认地址后，下单时自动带入该地址

### 流程11：地址簿选择流程（实物兑换场景）

```mermaid
flowchart TD
    subgraph 普通用户
        A([开始]) --> B[选择实物商品]
        B --> C[点击兑换]
        C --> D[弹出兑换表单]
        D --> E{有默认地址？}
        E -->|是| F[自动带入默认地址]
        E -->|否| G[表单为空]
        F --> H[查看/修改表单]
        G --> H
        H --> I[点击选择地址簿]
        I --> J[弹出地址列表]
        J --> K[选择地址]
        K --> L[自动回显地址]
        L --> H
        H --> M[确认兑换信息]
        M --> N[提交订单]
        N --> O([结束])
    end
```

**说明**：
- 实物商品兑换时，表单自动带入默认地址（若有）
- 用户可点击「选择地址簿」按钮，二次弹框从地址列表中选择
- 选择后地址自动回显到表单
- 用户也可手动修改表单中的地址信息
