# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

系统为所有用户提供统一的站内消息通知能力，支持订单超时提醒、审核结果通知、密码重置通知等业务场景的消息推送。消息类型完全可配置，便于后续扩展新场景。每位用户只能查看自己收到的消息，确保数据隐私。

### 1.2 本功能业务目标

1. 为系统提供统一的站内消息推送机制，支持业务事件自动触发
2. 为用户提供便捷的消息查看和管理能力，提升用户体验
3. 通过可配置的消息类型和触发机制，支持业务灵活扩展

## 3. 页面与功能总览

### 3.1 页面清单


**用户可操作功能：**

  - 触发按钮：标记已读

  - 触发方式：点击按钮

## 5. 非页面功能详细说明

### 5.1 消息发送服务

#### 5.1.1 功能概述

**触发方式：**API调用 / 事件监听  
**触发时机：**其他业务模块在业务发生时调用消息服务发送消息  
**功能职责：**为其他模块提供消息发送能力，支持可配置的消息类型和内容模板

#### 5.1.2 处理流程

1. **接收请求**：接收调用方传递的消息发送请求（接收者ID、消息类型、标题、内容等参数）
2. **参数校验**：校验必填参数（receiver_id、type、title、content）是否完整有效
3. **消息入库**：将消息写入数据库，状态为"未读"
4. **返回结果**：返回发送结果

#### 5.1.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| receiver_id | 调用方传入 | 消息接收者ID | 必填，标识消息发送给哪个用户 |
| type | 调用方传入/系统配置 | 消息类型 | 必填，用于分类和筛选 |
| title | 调用方传入 | 消息标题 | 必填，简短描述 |
| content | 调用方传入 | 消息内容 | 必填，完整消息文本 |
| status | 系统预设 | 消息状态 | 默认"未读" |

#### 5.1.4 业务规则

- **规则1：receiver_id必填**
  - 规则描述：调用方必须传入有效的接收者用户ID
  - 约束说明：无效或不存在的用户ID应拒绝发送

- **规则2：消息类型可配置**
  - 规则描述：消息类型在系统配置中管理，支持新增修改
  - 约束说明：调用方传入的类型必须在系统配置中存在

- **规则3：数据隔离**
  - 规则描述：所有消息数据通过 shop_id 隔离
  - 约束说明：消息入库时自动带上当前用户的 shop_id

#### 5.1.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| receiver_id无效 | 拒绝发送 | 返回错误码和提示"接收者不存在" |
| 消息类型未配置 | 拒绝发送 | 返回错误码和提示"消息类型未配置" |
| 参数缺失 | 拒绝发送 | 返回错误码和提示"缺少必填参数" |

#### 5.1.6 与其他功能的关联

- **上游依赖：**无（消息服务是被调用方）
- **下游影响：**被订单超时提醒、审核结果通知、密码重置通知等业务场景调用

### 5.2 消息类型配置管理

#### 5.2.1 功能概述

**触发方式：**管理员操作  
**触发时机：**管理员需要在系统中新增或修改消息类型  
**功能职责：**为管理员提供消息类型的配置能力，支持动态添加新的消息类型

#### 5.2.2 处理流程

1. 管理员进入消息类型配置页面
2. 管理员可查看已有消息类型列表
3. 管理员可新增消息类型（填写类型编码、类型名称）
4. 管理员可编辑已有消息类型
5. 管理员可删除未使用中的消息类型

#### 5.2.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| 类型编码 | 管理员输入 | 唯一标识 | 必填，用于代码中引用 |
| 类型名称 | 管理员输入 | 显示名称 | 必填，用于界面展示 |

#### 5.2.4 业务规则

- **规则1：类型编码唯一**
  - 规则描述：同一店铺下类型编码不能重复
  - 约束说明：新增时校验，重复则拒绝

- **规则2：内置类型保护**
  - 规则描述：系统内置的消息类型不可删除
  - 约束说明：删除时校验，如果是内置类型则提示不可删除

#### 5.2.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| 类型编码重复 | 拒绝操作 | 返回错误提示"类型编码已存在" |
| 删除内置类型 | 拒绝操作 | 返回错误提示"内置类型不可删除" |

#### 5.2.6 与其他功能的关联

- **上游依赖：**无
- **下游影响：**消息发送服务依赖此配置判断消息类型是否有效
