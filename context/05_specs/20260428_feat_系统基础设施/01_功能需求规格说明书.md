# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

系统基础设施是多店铺积分商城系统的底层支撑能力，为所有业务模块提供**数据隔离**和**文件存储**两大核心能力。任何业务模块的正常运行都依赖这两个基础设施模块。

**数据隔离**：系统采用表级别租户隔离架构，通过 shop_id 字段区分不同店铺的数据，实现店铺间数据完全隔离。

**文件存储**：系统提供统一的文件存储服务，支持本地存储（可扩展OSS），其他业务模块通过 file_id 关联文件。

### 1.2 本功能业务目标

1. **租户数据隔离**：通过 shop_id 自动拦截过滤，确保店铺用户只能访问本店数据，普通用户只能访问本店上架商品和自己的订单
2. **统一文件服务**：提供独立的文件上传下载接口，其他模块通过 file_id 关联，无需关心存储实现细节
3. **存储扩展性**：存储方式通过后端配置文件管理，重启后生效，便于后续扩展云存储（如OSS）

## 2. 角色与权限矩阵

> **说明**：本章节整合所有权限信息，是权限控制的**单一真实来源**。第4章各页面的权限要求均引用本章节，避免重复定义。

### 2.1 数据权限（行级访问控制）

> 定义不同角色针对**本功能数据**的行级别访问权限（即"谁能看到哪些数据行"）。

| 角色 | 数据范围 | 说明 |
|------|---------|------|
| 店铺用户 | 本店所有数据 | 通过 shop_id 自动过滤，仅返回 shop_id=当前用户所属店铺的数据 |
| 普通用户 | 本店上架商品 + 自己订单 | 商品列表过滤 shop_id=当前店铺 AND status=已上架；订单过滤 shop_id=当前店铺 AND user_id=当前用户 |
| 平台管理员 | 全部数据 | 无 shop_id 过滤，可访问所有数据 |

**数据范围说明：**
- **仅本店**：所有查询自动附加 `shop_id = 当前用户所属店铺ID` 条件
- **全部数据**：平台管理员查询无 shop_id 过滤
- **自定义**：根据用户类型动态计算过滤条件（店铺用户 vs 普通用户）

### 2.2 页面访问权限

> 定义不同角色对各页面的访问和操作权限。

**说明**：系统基础设施为底层模块，不提供独立管理页面。数据隔离和文件存储通过 AOP/拦截器对所有业务模块透明生效。

| 页面/接口 | 可访问角色 | 可操作角色 | 字段级权限（如有） |
|---------|----------|----------|------------------|
| 数据隔离框架 | 所有角色 | 平台管理员（配置权限） | 通过AOP自动生效 |
| 文件存储接口 | 所有角色 | 调用方模块 | file_id关联权限由调用方控制 |

**权限说明：**
- **数据隔离**：通过 AOP 拦截器自动注入 shop_id 过滤条件，开发者无感知
- **文件存储**：提供统一上传下载接口，权限由调用方业务模块控制

## 5. 非页面功能详细说明

### 5.1 数据隔离框架

#### 5.1.1 功能概述

**触发方式：** AOP拦截器
**触发时机：** 所有Service层数据库查询方法执行前
**功能职责：** 自动注入shop_id过滤条件，实现表级别租户数据隔离

#### 5.1.2 处理流程

1. **拦截请求**：AOP拦截所有Service层带有数据库查询的方法
2. **获取当前用户上下文**：从SecurityContext获取当前用户信息（userId, shopId, role）
3. **判断用户角色**：
   - 平台管理员：无shop_id过滤
   - 店铺用户：添加shop_id=当前店铺过滤
   - 普通用户：根据查询对象类型添加不同过滤条件
4. **注入过滤条件**：将shop_id条件动态注入到SQL查询中
5. **执行查询**：带着过滤条件执行原查询

#### 5.1.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| shop_id | 用户会话 | 数据隔离标识 | 区分不同店铺的数据 |
| user_id | 用户会话 | 普通用户订单过滤 | 确保用户只能看自己的订单 |
| user_role | 用户会话 | 权限判断 | 决定过滤策略 |

#### 5.1.4 业务规则

- **规则1：店铺用户数据隔离**
  - 规则描述：店铺用户所有数据查询自动附加 `shop_id = 当前用户所属店铺ID` 条件
  - 触发条件：店铺用户发起任何数据查询
  - 约束说明：店铺用户无法跨店查询数据

- **规则2：普通用户数据隔离**
  - 规则描述：普通用户查询商品时过滤 `shop_id = 当前店铺 AND status = 已上架`；查询订单时过滤 `shop_id = 当前店铺 AND user_id = 当前用户ID`
  - 触发条件：普通用户发起商品列表或订单列表查询
  - 约束说明：普通用户无法查看他人订单，无法查看未上架商品

- **规则3：平台管理员全局访问**
  - 规则描述：平台管理员查询无shop_id过滤，可访问所有店铺数据
  - 触发条件：平台管理员角色发起数据查询
  - 约束说明：仅限平台管理员角色

#### 5.1.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| 用户上下文缺失 | 拒绝访问 | 返回"用户未登录" |
| shop_id为空（店铺用户） | 拒绝访问 | 返回"店铺信息缺失" |
| SQL注入尝试 | 过滤特殊字符 | 记录安全日志 |

#### 5.1.6 与其他功能的关联

- **上游依赖**：用户认证模块（提供用户上下文）
- **下游影响**：所有业务模块的数据查询自动受此框架影响

### 5.2 文件存储服务

#### 5.2.1 功能概述

**触发方式：** API接口调用
**触发时机：** 业务模块调用文件上传/下载接口
**功能职责：** 提供统一的文件存储和访问能力，支持本地存储（可扩展OSS）

#### 5.2.2 处理流程

**文件上传流程：**
1. 业务模块调用文件上传接口，提交文件流和元数据
2. 校验文件类型、文件大小（默认10MB）
3. 生成唯一文件标识（UUID）+ 时间戳目录结构
4. 保存文件到存储介质（本地或OSS）
5. 创建文件记录（file_id, original_name, stored_name, path, size, type, upload_time）
6. 返回 file_id 给调用方

**文件下载流程：**
1. 业务模块传入 file_id
2. 根据 file_id 查询文件记录
3. 校验文件是否存在
4. 从存储介质读取文件
5. 返回文件流给调用方

**文件删除流程：**
1. 业务模块传入 file_id（仅当关联业务删除时）
2. 根据 file_id 查询文件记录
3. 物理删除存储介质中的文件
4. 删除文件记录

#### 5.2.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| file_id | 系统生成 | 文件唯一标识 | UUID，主键 |
| original_name | 用户上传 | 原始文件名 | 展示用 |
| stored_name | 系统生成 | 存储文件名 | UUID生成 |
| storage_path | 系统生成 | 存储路径 | 包含目录结构 |
| file_size | 用户上传 | 文件大小 | 单位：字节 |
| mime_type | 用户上传 | MIME类型 | 如：image/png |
| shop_id | 用户会话 | 租户隔离 | 文件归属店铺 |
| uploader_id | 用户会话 | 上传用户 | 记录上传人 |
| upload_time | 系统生成 | 上传时间 | 时间戳 |

#### 5.2.4 业务规则

- **规则1：文件类型限制**
  - 规则描述：仅允许上传图片类型文件（jpg, jpeg, png, gif, webp）和文档类型（pdf, doc, docx, xls, xlsx）
  - 触发条件：文件上传时
  - 约束说明：不在白名单内的文件类型拒绝上传

- **规则2：文件大小限制**
  - 规则描述：单文件大小不超过10MB
  - 触发条件：文件上传时
  - 约束说明：超过大小返回错误提示

- **规则3：存储位置配置**
  - 规则描述：存储介质类型（本地/OSS）通过后端配置文件指定，重启后生效
  - 触发条件：系统启动或文件操作时
  - 约束说明：配置变更需重启服务

- **规则4：文件记录物理删除**
  - 规则描述：文件删除时同时删除存储介质中的文件和数据库记录
  - 触发条件：调用方显式删除关联文件时
  - 约束说明：不提供回收站功能，删除不可逆

- **规则5：file_id关联模式**
  - 规则描述：业务模块仅存储file_id，不存储文件本身。获取文件时通过file_id查询
  - 触发条件：业务模块需要展示或下载文件时
  - 约束说明：file_id必须在文件存在时才有意义

#### 5.2.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| 文件类型不支持 | 拒绝上传 | 返回"不支持的文件类型" |
| 文件大小超限 | 拒绝上传 | 返回"文件大小超过限制" |
| file_id不存在 | 返回空 | 返回404 |
| 存储介质故障 | 返回错误 | 返回"文件服务异常" |
| OSS扩展失败 | 降级到本地 | 记录错误日志 |

#### 5.2.6 与其他功能的关联

- **上游依赖**：用户认证模块（提供上传用户上下文）
- **下游影响**：被商品管理（商品图片）、店铺运营（店铺Logo）等模块调用
