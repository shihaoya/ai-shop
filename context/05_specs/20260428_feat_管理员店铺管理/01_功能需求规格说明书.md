# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

为平台管理员提供店铺审核与管理能力，确保店铺合法合规运营。管理员可以查看所有店铺申请、处理审核请求、对违规店铺执行冻结操作。

### 1.2 本功能业务目标

1. 管理员可查看所有店铺的列表和状态
2. 管理员可审核待处理的店铺申请（通过/拒绝）
3. 管理员可对违规店铺执行冻结/解冻操作
4. 管理员可查看店铺详情和历史申请记录

### 4.1 页面：店铺管理列表页

#### 4.1.1 页面概述

**页面形式：** 独立页面

**页面职责：** 展示所有店铺的列表，支持按状态筛选，管理员可执行审核、冻结、解冻等操作。

#### 4.1.2 涉及字段

##### A. 查询字段（列表页专用）

| 字段名称 | 字段类型 | 数据来源 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|--------|--------|
| 状态筛选 | 下拉多选 | 用户输入 | 可多选 | 全部 | 筛选特定状态的店铺 |
| 关键词搜索 | 文本 | 用户输入 | 支持模糊匹配 | 空 | 按店铺名称搜索 |

##### B. 显示字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 店铺ID | 文本 | 数据库查询 | 页面加载 | 无 | 系统生成 | 店铺唯一标识 |
| 店铺名称 | 文本 | 数据库查询 | 页面加载 | 无 | 无 | 店铺名称 |
| Logo | 图片 | 数据库查询 | 页面加载 | 无 | 默认图标 | 店铺Logo |
| 简介 | 文本 | 数据库查询 | 页面加载 | 无 | 空 | 店铺简介（最多100字符） |
| 状态 | 标签 | 数据库查询 | 页面加载 | 无 | 无 | 店铺当前状态：待审核/已通过/已拒绝/已冻结 |
| 申请时间 | 日期时间 | 数据库查询 | 页面加载 | 无 | 无 | 店铺提交申请的时间 |
| 操作人 | 文本 | 数据库查询 | 页面加载 | 无 | 无 | 最后处理该店铺的管理员 |
| 操作时间 | 日期时间 | 数据库查询 | 页面加载 | 无 | 无 | 最后操作时间 |

#### 4.1.3 功能与按钮

**用户可操作功能：**

- **查看详情**
  - 触发按钮：详情
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：管理员可操作
  - 加载状态：点击后显示loading
  - 功能说明：弹出详情弹窗，显示店铺完整信息（名称、Logo、简介、状态、申请时间、审核信息等）。无数据时显示空状态。

- **查看历史申请记录**
  - 触发按钮：历史申请记录
  - 按钮级别：行级
  - 按钮位置：每行操作列
  - 触发方式：点击按钮
  - 权限要求：管理员可操作
  - 加载状态：点击后显示loading
  - 功能说明：弹出历史申请记录弹窗，显示该店铺的历史申请/审核/冻结/解冻记录列表，按时间倒序排列。无记录时显示空状态。

- **审核店铺**
  - 触发按钮：审核
  - 按钮级别：行级
  - 按钮位置：每行操作列（仅待审核状态显示）
  - 触发方式：点击按钮
  - 权限要求：管理员可操作
  - 加载状态：点击后显示loading
  - 功能说明：弹出审核操作弹窗，管理员可选择「通过」或「拒绝」，拒绝时必须填写拒绝理由。通过后店铺状态变为已通过，拒绝后变为已拒绝。

- **冻结店铺**
  - 触发按钮：冻结
  - 按钮级别：行级
  - 按钮位置：每行操作列（仅已通过状态显示）
  - 触发方式：点击按钮
  - 权限要求：管理员可操作
  - 加载状态：点击后显示loading
  - 功能说明：弹出冻结确认弹窗，管理员确认后执行冻结操作。冻结后店铺状态变为已冻结，店铺用户不能操作商品（商品自动下架）。

- **解冻店铺**
  - 触发按钮：解冻
  - 按钮级别：行级
  - 按钮位置：每行操作列（仅已冻结状态显示）
  - 触发方式：点击按钮
  - 权限要求：管理员可操作
  - 加载状态：点击后显示loading
  - 功能说明：弹出解冻确认弹窗，管理员确认后执行解冻操作。解冻后店铺状态恢复为已通过。

**后台自动流程（如有）：**

无

#### 4.1.4 业务规则

- **规则1：店铺冻结影响**
  - 规则描述：店铺被冻结后，该店铺下的商品自动下架，店铺用户不能进行商品和用户管理操作
  - 触发条件：管理员执行冻结操作后
  - 约束说明：解冻后可恢复

- **规则2：冻结状态不可删除**
  - 规则描述：状态为已冻结的店铺不允许被删除
  - 触发条件：删除操作时检查
  - 约束说明：仅已通过、已拒绝状态可删除

#### 4.1.5 交互逻辑

- 列表加载时显示骨架屏或loading状态
- 无数据时显示空状态提示（"暂无店铺数据"）
- 操作成功后显示Toast提示（"操作成功"）
- 操作失败后显示错误提示（"操作失败：原因"）
- 弹窗关闭后自动刷新列表数据

#### 4.1.6 权限要求

参见第2.2节"页面访问权限"，本页面的可访问角色、可操作角色、字段级权限均以第2章定义为准。

## 6. 数据状态定义

### 6.1 数据状态

| 状态名称 | 状态说明 | 可转换到的状态 |
|---------|---------|--------------|
| 待审核 | 店铺已提交申请，等待管理员审核 | 已通过、已拒绝 |
| 已通过 | 审核通过，店铺正常运营 | 已冻结 |
| 已拒绝 | 审核未通过，店铺无法运营 | 无（需重新申请） |
| 已冻结 | 因违规被管理员冻结，店铺无法运营 | 已通过（解冻后） |

### 6.2 状态转换规则

| 当前状态 | 可转换到的状态 | 转换条件 | 转换触发方式 |
|---------|------------|---------|-------------|
| 待审核 | 已通过 | 管理员审核通过 | 用户操作（审核弹窗） |
| 待审核 | 已拒绝 | 管理员审核拒绝（需填写拒绝理由） | 用户操作（审核弹窗） |
| 已通过 | 已冻结 | 管理员执行冻结操作 | 用户操作（冻结确认弹窗） |
| 已冻结 | 已通过 | 管理员执行解冻操作 | 用户操作（解冻确认弹窗） |
