# 功能需求规格说明书

## 1. 概述

### 1.1 功能背景

站内消息是系统内的消息通知模块，负责向用户推送系统自动产生的消息通知。用户在各业务模块中发生关键行为时（如订单状态变更、积分变动、系统公告等），系统通过消息发送接口自动向用户推送消息，用户可在"我的消息"页面集中查看所有消息。

### 1.2 本功能业务目标

1. **消息聚合**：将分散在各个业务模块的通知消息统一聚合到站内消息系统
2. **已读管理**：支持用户标记消息为已读，帮助用户跟踪未读消息
3. **数据隔离**：确保用户只能查看自己的消息，保障消息隐私

**权限说明：**
- **可访问角色**：已登录的系统用户
- **可操作角色**：登录用户均可操作（标记已读、一键已读）

## 3. 页面与功能总览

### 3.1 页面清单

| 序号 | 页面名称 | 页面形式 | 职责说明 | 包含功能 |
|-----|---------|---------|---------|---------|
| 1 | 我的消息 | 独立页面 | 展示用户收到的所有站内消息列表 | 查看消息列表、标记已读、一键已读 |

### 3.2 页面跳转流程

```
[入口：顶部导航"消息"图标] → [我的消息列表页]
                                    ↓
                              点击"[标记已读]" → [本条消息状态变为已读]
                                    ↓
                              点击"[一键已读]" → [所有未读消息状态变为已读]
```

**流程说明：**
1. 用户点击顶部导航的"消息"图标进入"我的消息"列表页
2. 用户可以查看所有收到的消息（按创建时间倒序排列）
3. 用户可以点击单条消息的"标记已读"按钮标记该消息为已读
4. 用户可以点击"一键已读"按钮将所有未读消息标记为已读

### 4.1 页面 1：我的消息

#### 4.1.1 页面概述

**页面形式：**独立页面

**页面职责：**展示当前用户收到的所有站内消息列表，支持按已读/未读状态筛选，支持标记单条或批量标记已读操作。

#### 4.1.2 涉及字段

##### A. 查询字段（列表页专用）

| 字段名称 | 字段类型 | 数据来源 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|--------|--------|
| 消息状态 | 下拉选择 | 系统预设 | 必填 | 全部 | 筛选消息的已读/未读状态 |

**消息状态选项：**
- 全部
- 未读
- 已读

##### B. 显示字段

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 消息标题 | 文本 | 数据库查询 | 页面加载 | 必填，最大200字符 | - | 消息的标题 |
| 消息内容 | 文本 | 数据库查询 | 页面加载 | 必填，最大2000字符 | - | 消息的正文内容 |
| 已读状态 | 标签 | 数据库查询 | 页面加载 | - | 未读 | 消息的已读/未读状态 |
| 创建时间 | 日期时间 | 数据库查询 | 页面加载 | - | - | 消息的创建时间 |

#### 4.1.3 功能与按钮

**用户可操作功能：**

- **一键已读**
  - 触发按钮：一键已读
  - 按钮级别：页面级
  - 按钮位置：列表顶部工具栏右侧
  - 触发方式：点击按钮
  - 权限要求：所有登录用户
  - 加载状态：点击后显示 loading，成功后刷新列表
  - 功能说明：点击后将当前筛选条件下所有未读消息标记为已读。操作成功后刷新列表，已读状态更新。

- **标记已读**
  - 触发按钮：标记已读
  - 按钮级别：行级
  - 按钮位置：每行消息的操作列
  - 触发方式：点击按钮
  - 权限要求：所有登录用户
  - 加载状态：点击后显示 loading，成功后更新该行已读状态
  - 功能说明：点击后将本条消息标记为已读。操作成功后该行"未读"标签变为"已读"。

#### 4.1.4 业务规则

- **规则1：消息列表默认按创建时间倒序排列**
  - 规则描述：最新的消息显示在列表最上方
  - 触发条件：页面加载时、刷新列表时
  - 约束说明：无

- **规则2：已读状态仅能从未读变更为已读，不可逆**
  - 规则描述：用户只能将消息标记为已读，不能将已读消息改回未读
  - 触发条件：用户点击"标记已读"或"一键已读"时
  - 约束说明：无

- **规则3：消息状态筛选仅影响列表展示，不改变数据**
  - 规则描述："未读"筛选仅显示 `is_read = false` 的消息；"已读"筛选仅显示 `is_read = true` 的消息；"全部"显示所有消息
  - 触发条件：用户切换消息状态下拉选择时
  - 约束说明：筛选仅控制展示，不触发数据变更

#### 4.1.5 交互逻辑

- **交互逻辑1：空状态提示**
  - 当列表为空时，显示空状态插图和文案："暂无消息"
  - 插图：消息图标
  - 文案："暂无消息"

- **交互逻辑2：列表排序规则**
  - 消息列表按创建时间倒序排列（最新的在最上方）

- **交互逻辑3：按钮禁用规则**
  - "一键已读"按钮：当筛选结果中无未读消息时，按钮置灰禁用
  - "标记已读"按钮：当该条消息已是已读状态时，按钮不显示或置灰

#### 4.1.6 权限要求

> **参见第2.2节"页面访问权限"**，本页面的可访问角色、可操作角色、字段级权限均以第2章定义为准。

**本页面特殊权限说明（如有）：**
- 无特殊权限，所有登录用户权限相同

## 5. 非页面功能详细说明

### 5.1 消息发送接口

#### 5.1.1 功能概述

**触发方式：** API调用
**触发时机：** 其他业务模块调用消息发送接口时
**功能职责：** 接收其他模块的发送消息请求，创建站内消息记录

#### 5.1.2 处理流程

1. **接收请求**：其他业务模块调用消息发送接口，传入 `user_id`（接收消息的用户ID）、`title`（消息标题）、`content`（消息内容）
2. **参数校验**：校验 `user_id` 必填且存在，`title` 必填且不超过200字符，`content` 必填且不超过2000字符
3. **创建消息**：在消息表中插入一条新记录，`is_read = false`，`create_time = 当前时间`
4. **返回结果**：返回创建成功后的消息ID

#### 5.1.3 涉及数据

| 数据项 | 数据来源 | 数据用途 | 说明 |
|-------|---------|---------|------|
| user_id | 调用方传入 | 消息接收者 | 必填，指向用户表ID |
| title | 调用方传入 | 消息标题 | 必填，最大200字符 |
| content | 调用方传入 | 消息内容 | 必填，最大2000字符 |
| is_read | 系统默认值 | 已读状态 | 默认 false |
| create_time | 系统自动 | 创建时间 | 默认当前时间戳 |

#### 5.1.4 业务规则

- **规则1：消息创建后不可修改、不可删除**
  - 站内消息为只读消息，创建后不允许修改内容，不允许删除

- **规则2：消息发送是同步操作**
  - 调用方发起消息发送请求后，同步等待消息创建完成并返回结果

#### 5.1.5 异常处理

| 异常场景 | 处理方式 | 错误提示/日志 |
|---------|---------|--------------|
| user_id 不存在 | 返回错误 | code: 400, message: "用户不存在" |
| title 超过200字符 | 返回错误 | code: 400, message: "消息标题不能超过200字符" |
| content 超过2000字符 | 返回错误 | code: 400, message: "消息内容不能超过2000字符" |
| 数据库写入失败 | 返回错误 | code: 500, message: "消息发送失败" |

#### 5.1.6 与其他功能的关联

- **上游依赖：**
  - 用户模块（通过 user_id 关联验证用户存在性）
- **下游影响：**
  - 我的消息页面（展示消息列表）
